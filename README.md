This project is a CLI text editor in C. The document itself is stored as a doubly linked list, where each node is one line of text — this lets me insert or delete lines without shifting anything else. For undo/redo, I use two stacks: whenever I insert or delete, I record that action and push it onto the undo stack. Undo pops the most recent action, reverses it on the linked list, and pushes it onto the redo stack — so redo can pop it back and reapply it. Any brand-new edit clears the redo stack, since that 'future' history becomes invalid. I also added save/load so the document persists to a text file between runs
